🔐 Secure Handshake

In-person trades,
without the risk.

Every local specimen handover uses a verification system that makes PIN spoofing, front-running, and replay attacks impossible — so funds only release when the physical handover is confirmed.

01
Buyer Creates a Verification Code

Before meeting, the buyer generates a secret PIN and unique salt. These are combined and hashed — only the resulting commitment is transmitted to the platform. The raw PIN never leaves the buyer's device.

Commitment Hash
keccak256(abi.encodePacked(pin, salt, buyerAddress))
= 0x8f4a2c3d9e1b7f…
02
Payment Locks in Escrow

Payment is held in a secure escrow account. Neither party can access the funds until the verified handover step completes successfully.

✓ Escrow Protected ✓ Anti-Re-entry Guard
03
QR Scan at Pickup

At the physical handover, the buyer presents a QR code. You scan it — no typing, no delays. The platform checks the code against the stored commitment instantly.

📱 { purchaseId: 47, pin: 8391, salt: "0xf4a2…" }
04
Verified — Escrow Released

The platform validates the code against the stored commitment. If it matches — escrow releases instantly and the sale is complete. If it doesn't — the transaction is rejected. No exceptions, no disputes.
